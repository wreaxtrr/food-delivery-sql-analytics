\# Концептуальная модель базы данных



\## Описание проекта



Проект представляет собой базу данных сервиса доставки еды.



База данных хранит информацию о пользователях, ресторанах, блюдах, заказах, курьерах, платежах, промокодах и событиях пользователей в приложении.



Цель проекта — спроектировать структуру базы данных и реализовать аналитические SQL-запросы для расчёта продуктовых и операционных метрик.



\## Основные сущности



\### users



Пользователи сервиса доставки еды.



\### restaurants



Рестораны, из которых пользователи могут делать заказы.



\### couriers



Курьеры, доставляющие заказы пользователям.



\### categories



Категории блюд.



\### products



Блюда или товары, доступные для заказа в ресторанах.



\### orders



Заказы пользователей.



\### order\_items



Позиции внутри заказа.



\### payments



Платежи по заказам.



\### promocodes



Промокоды, которые могут применяться к заказам.



\### order\_status\_history



История изменения статусов заказа.



\### app\_events



События пользователей в приложении: просмотр ресторана, просмотр товара, добавление в корзину, оформление заказа.



\## Связи между сущностями



\- Один пользователь может сделать много заказов.

\- Один ресторан может получить много заказов.

\- Один ресторан может иметь много товаров.

\- Один заказ может содержать много позиций.

\- Один товар может встречаться во многих заказах.

\- Один курьер может доставить много заказов.

\- Один заказ имеет один платёж.

\- Один промокод может использоваться во многих заказах.

\- Один заказ может иметь несколько записей в истории статусов.

\- Один пользователь может совершить много событий в приложении.
